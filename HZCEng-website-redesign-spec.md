# H&Z Consulting Engineering Website Redesign

## Design Goal

Reposition H&Z Consulting Engineering as a polished, engineer-led structural firm for serious residential, commercial, renovation, and inspection work across NY, NJ, and CT. The redesigned site should feel precise, trustworthy, modern, and easy to act on.

The current brand blue stays central, but the site should move away from washed-out imagery and sparse brochure sections toward a premium architectural presentation with strong hierarchy, real proof, and repeated consultation paths.

## Brand System

### Color Palette

- Primary blue: `#0879B8`  
  Use for main CTAs, active navigation, key headings, and small proof accents.
- Deep navy: `#08273A`  
  Use for body text, footer, high-contrast sections, and premium CTA panels.
- Structural slate: `#425563`  
  Use for secondary text and technical captions.
- Soft steel: `#E8EEF2`  
  Use for section backgrounds, card shells, and blueprint-style dividers.
- Warm white: `#FBFCFD`  
  Use as the main page background.
- Mist blue: `#D9ECF6`  
  Use sparingly behind trust badges, icons, and selected callouts.
- Alert/accent gold: `#C79A3B`  
  Use very sparingly for “licensed,” “inspection,” or “priority” details. Do not let it compete with blue.

### Typography

- Headings: premium geometric sans, such as `Plus Jakarta Sans`, `Sora`, `Avenir Next`, or `Geist`.
- Body: clean readable sans, such as `Source Sans 3`, `Inter`, `Aptos`, or `IBM Plex Sans`.
- Heading style: large, confident, slightly tight tracking, generous line height.
- Body style: short paragraphs, 16-18px desktop, 15-16px mobile, never stretched wider than 680px.

Suggested scale:

- Hero H1: 64-78px desktop, 42-48px tablet, 34-40px mobile.
- Section H2: 44-56px desktop, 32-40px mobile.
- H3/card titles: 20-26px.
- Body: 16-18px.
- Captions/eyebrows: 11-13px uppercase with wide letter spacing.

### Visual Motifs

- Use subtle blueprint grid lines, structural section marks, CAD-style labels, and thin technical dividers.
- Use real project imagery as the hero of the brand. Avoid low-contrast text over faded city backgrounds.
- Use refined rounded rectangles, not playful blobs. Radii should feel architectural: 18px, 24px, 32px.
- Use soft depth through layered panels and light borders, not heavy shadows.

## Components

### Header

Use a floating white/navy-tinted header with the logo on the left, concise nav links in the center/right, and a clear CTA button.

Desktop nav:

- Home
- Services
- Projects
- About
- Contact
- CTA: `Request Consultation`

Mobile nav:

- Logo left
- Menu button right
- Sticky bottom contact bar optional: `Call` and `Request Consultation`

### Buttons

Primary button:

- Filled H&Z blue
- White text
- Pill shape
- Right-side small circular arrow capsule
- Hover: slightly darker blue, arrow moves 2-4px

Secondary button:

- White or transparent background
- Navy text
- Soft blue border
- Hover: mist-blue fill

CTA labels:

- `Request a Free Consultation`
- `Discuss a Project`
- `View Recent Projects`
- `Call (718) 569-8677`

### Cards

Use cards as “engineered panels”:

- Outer light steel shell
- Inner warm white or navy surface
- Thin hairline border
- Small technical label, for example `SERVICE 01`
- Clear title, one-sentence value proposition, 2-3 use cases

Do not use a uniform grid of identical boxes for the whole page. Mix feature cards, split sections, large image panels, and proof strips.

## Builder-Ready Implementation Guide

### Global Styling

- Set the page background to warm white.
- Limit content width to roughly 1180-1240px.
- Use generous vertical spacing: 96-140px desktop, 64-88px tablet, 48-72px mobile.
- Keep body text line length under 680px.
- Add subtle blueprint/grid texture only in hero or process areas, never behind dense copy.

### Header

- Use a white floating header with 80-90% opacity if the platform supports blur; otherwise use solid white.
- Add a small blue CTA button on desktop.
- On mobile, prioritize `Call` and `Request Consultation`.

### Hero

- Replace the faded skyline hero with a clean split layout.
- Use a high-quality project image, render, or structural model in a framed card.
- Put CTA buttons above the fold.
- Add the trust strip directly under the CTAs.

### Services

- Replace the existing icon grid with service cards.
- Keep icons if they are crisp, but pair each with meaningful text.
- Use 2-column or 3-column layout on desktop, 1-column on mobile.

### Projects

- Use large image cards rather than small thumbnails.
- Add project scope and engineering challenge to each card.
- If images vary in quality, use consistent aspect ratios and subtle overlays/captions.

### Contact

- Repeat contact CTA at least three times: hero, mid-page after why section, final CTA.
- Make phone and email clickable.
- Use one primary CTA label consistently: `Request a Free Consultation`.

### Accessibility

- Maintain strong contrast between blue text/buttons and backgrounds.
- Do not place small text over images.
- Buttons should have visible focus states.
- Image alt text should describe project content, not just “project image.”
- Mobile tap targets should be at least 44px tall.

### SEO Notes

- H1 should include “Structural Engineering” and service geography.
- Add clear service keywords naturally: structural design, engineering inspection, special inspection, renovations, additions, foundation, underpinning, New York, New Jersey, Connecticut.
- Add project pages or expanded case studies later for local SEO.
